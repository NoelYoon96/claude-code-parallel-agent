# 구현 검증

## 목적

프로젝트에 등록된 모든 `verify-*` 스킬을 순차적으로 실행하여 통합 검증을 수행합니다:

- 각 스킬의 Workflow에 정의된 검사를 실행
- 각 스킬의 Exceptions를 참조하여 false positive 방지
- 발견된 이슈에 대해 수정 방법을 제시
- 사용자 승인 후 수정 적용 및 재검증

## 워크플로우

### Step 1: 실행 대상 스킬 동적 탐색

```bash
ls .claude/skills/verify-*/SKILL.md 2>/dev/null
```

스킬이 0개인 경우 → `/manage-skills`를 실행하여 생성하세요.

### Step 2: 순차 실행

각 스킬의 SKILL.md를 읽고 Workflow 섹션의 검사를 순서대로 실행합니다:

1. 검사에 명시된 도구(Grep, Glob, Read, Bash)를 사용하여 패턴 탐지
2. PASS/FAIL 기준에 대조
3. Exceptions 섹션에 해당하는 패턴은 면제 처리
4. FAIL인 경우 이슈 기록 (파일 경로, 문제 설명, 수정 권장 사항)

### Step 3: 통합 보고서

모든 스킬 실행 완료 후 결과를 하나의 보고서로 통합합니다.

### Step 4: 사용자 액션 확인

이슈 발견 시:
1. **전체 수정** - 모든 권장 수정사항을 자동으로 적용
2. **개별 수정** - 각 수정사항을 하나씩 검토 후 적용
3. **건너뛰기** - 변경 없이 종료

### Step 5: 수정 적용

### Step 6: 수정 후 재검증

이슈가 있었던 스킬만 다시 실행하여 Before/After를 비교합니다.

## 예외사항

1. 등록된 스킬이 없는 프로젝트 — 안내 메시지 표시 후 종료
2. 스킬의 자체적 예외 — Exceptions 섹션에 정의된 패턴은 이슈로 보고하지 않음
3. verify-skill 자체 — 실행 대상에 자기 자신을 포함하지 않음
4. manage-skills — `verify-`로 시작하지 않으므로 실행 대상에 포함되지 않음
